# Charter Modeling Flow — Generic Operating Flow

## Overview

Charter works as a continuous flow between:

- observation
- investigation
- structure
- legitimacy
- monitoring

The flow is generic enough to support:

- personal use
- organizational decision-making
- scientific research
- investigations
- real-time systems

It is not a rigid pipeline.  
It is a loop with explicit boundaries.

---

# 1. Start with Context

Everything begins with a **context boundary**.

This may be:

- an Area
- an Epic
- an identity-scoped slice
- a target set
- a manually chosen subset of structure

Purpose:

- define what part of reality you are looking at
- keep work bounded
- avoid accidental scope creep

At this stage, structure may still be flat.

---

# 2. Bring in What You Know

Charter can begin from different starting points:

### A. Existing decisions
- current resolutions
- previously accepted structure

### B. Observations
- signals
- requests
- supportability concerns
- metrics
- human feedback

### C. External structure
- foreign Areas
- imported bundles
- discovered artifacts

### D. Blank investigation
- start from nothing
- create Items directly

This step is about assembling candidate reality, not deciding yet.

---

# 3. Create or Open an Investigation

Once context exists, work moves into an investigation space.

This is where you:

- ask questions
- collect observations
- create hypotheses
- connect related things
- surface candidate relationships

The investigation may:

- start blank
- start from observations
- start from existing resolutions projected into Items
- start from foreign structure

This is the main workspace for deliberate thinking.

---

# 4. Project Existing Structure When Needed

If you want to reason from current decisions, Charter can project legitimate structure into investigation.

This means:

- select existing resolutions
- create resolution-derived Items
- preserve lineage with `derived_from`

Purpose:

- simulate without mutating legitimacy
- explore alternatives safely
- build on real structure without copying authority

This is the main “rebase from reality” move.

---

# 5. Explore and Reshape

Inside investigation, you can:

- refine Items
- split Items
- merge Items
- group Items
- connect Items to existing structure
- surface candidate relationships

This is also where structural modeling happens.

You may:

- keep things flat
- decompose broader ideas into narrower ones
- synthesize narrower ideas into broader ones
- reshape the graph before anything becomes real

Nothing here is legitimate yet.

---

# 6. Monitor While You Work

Investigations may remain connected to ongoing observations.

You can subscribe to feeds or scoped monitoring surfaces so that:

- relevant changes keep arriving
- investigations stay current
- long-running work remains grounded in reality

This is especially useful for:

- care loops
- operations
- scientific work
- long-lived organizational issues

Monitoring provides context.  
It does not create decisions automatically.

---

# 7. Stabilize Candidate Outputs

As understanding improves, some Items become stable enough to preserve.

These may become:

- READY
- LOCKED
- SETTLED
- APPLIED later through legitimacy

This step is about saying:

- this is worth preserving
- this is worth reviewing
- this is stable enough to compare against reality

Stability is not authority.

---

# 8. Reconcile with Reality

Before promoting anything, Charter may reconcile investigation with current structure.

This includes:

- checking whether Items are already satisfied by existing resolutions
- identifying drift between investigation and legitimacy
- pulling updated resolution state into investigation
- tracing lineage between Items and Resolutions

This is the bridge between:

- what we are thinking
- what the system currently recognizes as real

---

# 9. Promote What Should Become Real

When candidate Items or relationships are ready, they move through reconciliation toward legitimacy.

This is where Charter decides:

- what becomes a resolution
- what remains investigative
- what relationships become durable
- what gets rejected, deferred, or abandoned

Only this stage can produce legitimate structural outcomes.

Nothing becomes legitimate just because it exists in investigation.

---

# 10. Preserve Lineage

Every meaningful change should preserve traceability.

Charter keeps track of:

- what came from what
- what was reviewed
- what became legitimate
- what stayed exploratory
- what was synchronized back into investigation

This makes it possible to:

- understand history
- revisit old reasoning
- compare alternatives
- evolve structure without rewriting the past

---

# 11. Observe the Result

Once decisions are in place, Charter continues observing what happens.

This allows you to see:

- whether decisions are holding
- where drift is forming
- where pressure is building
- where conflict or instability is emerging
- where improvement is happening

This is where the system starts to feel dynamic.

Over time, Charter can help make visible:

- tension
- pressure
- stability
- drift
- structural gravity
- emerging change

The goal is not to replace judgment.  
It is to make patterns easier to see.

---

# 12. Repeat

The flow is continuous:

context  
→ investigation  
→ reconciliation  
→ legitimacy  
→ monitoring  
→ renewed investigation

Charter is not a one-time decision system.

It is a system for evolving structure and understanding over time.

---

# Generic Flow Summary

A simplified version looks like this:

Context  
→ Gather structure / observations  
→ Investigate  
→ Project existing structure if needed  
→ Reshape and model  
→ Stabilize  
→ Reconcile  
→ Promote selected outputs  
→ Monitor results  
→ Repeat

---

# Why This Flow Matters

This flow makes it possible to:

- start small
- remain flexible
- preserve history
- simulate before committing
- connect human reasoning with structural rigor

It works whether you are:

- one person organizing ideas
- a team managing decisions
- a researcher testing hypotheses
- an operator monitoring systems
- an investigator working through ambiguity

The same flow applies.  
Only the scale changes.
