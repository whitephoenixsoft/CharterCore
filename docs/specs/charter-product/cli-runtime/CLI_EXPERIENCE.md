# Charter CLI — Experience & Operating Model

Status: FOUNDATIONAL (ENTRY LAYER)  
Applies to: Charter CLI (Primary Host Interface)  
Purpose: Define how interacting with Charter feels and behaves for users  

---

# 1. What the CLI Is

The Charter CLI is the primary way to interact with Charter.

It is not just a command runner.

It is a **working environment for thinking, deciding, and understanding over time**.

At its simplest, it can be used like:

- a decision journal  
- a structured notebook  
- a way to track ideas and outcomes  

Everything else builds on top of that.

---

# 2. The Core Experience

Using Charter CLI should feel:

- calm  
- predictable  
- explicit  
- safe  

You are never rushed.

You are never forced into structure.

You are never surprised by hidden changes.

Instead, you move through a simple pattern:

- observe  
- think  
- decide  
- reflect  

---

# 3. You Are Always Somewhere

Charter is context-aware.

At any moment, you are operating within:

- a **workspace** (your environment)  
- an **area** (your decision boundary)  

This is not something you constantly manage.

It is something the CLI keeps visible and stable.

You always know:

- where you are  
- what you are working on  
- what is currently active  

---

# 4. Explicit Over Implicit

Charter CLI does not guess.

- switching context is explicit  
- changing areas is explicit  
- making decisions is explicit  

Convenience exists, but it never overrides clarity.

If something changes, you chose it.

---

# 5. Thinking Comes First

Before decisions, there is space to think.

You can:

- write ideas  
- explore possibilities  
- connect thoughts  
- leave things unfinished  

Nothing here becomes permanent unless you choose it to.

This allows you to:

- explore without pressure  
- change direction freely  
- revisit ideas later  

---

# 6. Decisions Are Intentional

When something becomes a decision:

- it is explicit  
- it is traceable  
- it is preserved  

You do not “accidentally” create decisions.

You choose to make them real.

---

# 7. Work Happens in Flows

Some actions require structure.

For example:

- reviewing imported decisions  
- resolving conflicts  
- formalizing changes  

These are handled as **processes**.

Processes:

- guide you step by step  
- can be paused  
- can be resumed  
- never force completion  

You remain in control at all times.

---

# 8. One Active Change at a Time

Within a given area:

- only one active change process can exist at a time  

This keeps things:

- clear  
- consistent  
- safe  

If something else needs to happen:

- you pause the current process  
- then continue or start another  

Nothing overlaps silently.

---

# 9. You Can Always See the Present

The CLI makes it easy to understand what is happening now.

You can always see:

- current context and area  
- active processes  
- pending changes  
- important signals  

This is your **present state**.

---

# 10. You Can Always Revisit the Past

Nothing is lost.

You can inspect:

- past decisions  
- previous states  
- how things evolved  
- why something changed  

History is not hidden.

It is available when you need it.

---

# 11. Recovery Is Always Possible

If something goes wrong:

- your data is still there  
- your history is still intact  
- your structure is still visible  

You may need to:

- reorganize  
- reconcile  
- rebuild  

But you never start from nothing.

Mistakes are recoverable.

---

# 12. Structure Is Discoverable

You do not need to understand everything upfront.

You can begin with:

- simple notes  
- simple decisions  

Over time, you may discover:

- relationships between decisions  
- patterns in outcomes  
- structured workflows  
- deeper system behavior  

These are not required.

They are available when useful.

---

# 13. Guidance, Not Control

Charter may help you:

- notice conflicts  
- see similarities  
- identify gaps  
- understand structure  

This help is:

- optional  
- non-authoritative  
- non-intrusive  

You decide what to do.

---

# 14. Queries Are Always Available

You can ask questions about your system at any time.

For example:

- what is active  
- what changed  
- what is related  
- what is different  

These queries allow you to explore your system without changing it.

---

# 15. A Living System

Over time, your use of Charter becomes:

- richer  
- more structured  
- more informative  

What begins as a journal can become:

- a decision system  
- a coordination tool  
- a modeling environment  

But it never stops being understandable.

---

# 16. What the CLI Is Not

The CLI is not:

- a task manager  
- a chat system  
- an automation engine  
- a decision maker  

It does not:

- tell you what to do  
- enforce behavior  
- hide complexity  

It exposes structure.

---

# 17. Final Principle

The Charter CLI exists to give you:

- a clear place to think  
- a safe place to decide  
- a reliable way to understand change  

It helps you move from:

> ideas → decisions → understanding  

without losing clarity at any step.
